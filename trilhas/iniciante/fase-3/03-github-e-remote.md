# 03 — GitHub e Repositórios Remotos

Até agora aprendemos como o Git controla versões dentro do seu computador.

Mas projetos profissionais precisam de algo a mais:

Eles precisam ficar disponíveis na internet para:

* Servir como backup
* Permitir trabalho em equipe
* Compartilhar código com o mundo

É aí que entra o GitHub.

---

## ☁️ O que é GitHub?

GitHub é uma plataforma onde você pode armazenar seus projetos Git na nuvem.

Ele funciona como uma grande rede social de programadores e também como um local seguro para guardar seus códigos.

Com o GitHub você pode:

* Salvar projetos online
* Acessar seus projetos de qualquer computador
* Trabalhar com outras pessoas no mesmo projeto
* Mostrar seu portfólio para empresas

---

## 🗂️ O que é um Repositório Remoto?

Repositório é o local onde seu projeto fica armazenado.

Até agora, você viu o repositório local:

> Arquivos salvos apenas no seu computador

Quando usamos GitHub, criamos um repositório remoto:

> Uma cópia do projeto armazenada na internet

Esse repositório remoto mantém o mesmo histórico de versões do seu projeto local.

---

## 🔄 Como funciona na prática (conceitual)

Imagine o seguinte fluxo:

1️⃣ Você cria um projeto no seu computador
2️⃣ Faz alterações e salva versões com Git
3️⃣ Envia o projeto para o GitHub
4️⃣ Seu amigo baixa o projeto
5️⃣ Ele faz melhorias
6️⃣ As melhorias voltam para você

Assim, várias pessoas conseguem trabalhar juntas no mesmo projeto.

---

## 🎯 Por que isso é essencial no mercado?

Empresas precisam que:

* O código esteja seguro
* Equipes trabalhem juntas
* Projetos tenham backup
* Mudanças sejam rastreáveis

Por isso, hospedar projetos em plataformas como o GitHub é um padrão profissional.

---

## 🌍 GitHub como portfólio

Além de colaboração, o GitHub serve como vitrine profissional.

Recrutadores podem:

* Ver seus projetos
* Avaliar sua organização
* Analisar sua evolução como programador

Seu perfil no GitHub funciona como um currículo técnico.
