# 01 — Introdução ao Git

Imagine que você está escrevendo um trabalho importante e salva várias versões do arquivo:

trabalho-final.doc
trabalho-final-agora-vai.doc
trabalho-final-definitivo.doc
trabalho-final-definitivo-mesmo.doc

Fica confuso, difícil de organizar e fácil de perder algo importante.

O Git resolve esse problema.

---

## 📦 O que é Git?

Git é um **sistema de controle de versão**.

Ele registra o histórico das alterações feitas nos arquivos de um projeto, permitindo que você:

* Volte para versões anteriores
* Veja exatamente o que mudou
* Trabalhe com segurança sem medo de errar
* Desenvolva novas funcionalidades sem quebrar o que já funciona

Em vez de vários arquivos duplicados, você tem um **histórico organizado**.

---

## 🧠 Como o Git funciona (visão simples)

O Git tira “fotografias” do seu projeto ao longo do tempo.

Cada fotografia representa o estado dos arquivos naquele momento.

Essas fotografias são chamadas de **versões**.

Sempre que você salva uma nova versão, o Git registra:

* Quais arquivos mudaram
* O que foi alterado
* Quando mudou
* Quem fez a mudança

---

## 💻 Exemplo prático (conceitual)

Imagine um sistema de cadastro de clientes evoluindo ao longo do tempo.

### Versão 1

Sistema cadastra nome e telefone

### Versão 2

Sistema cadastra nome, telefone e email

### Versão 3

Sistema corrige erro no salvamento do telefone

Sem Git:
Você teria vários arquivos duplicados e confusos.

Com Git:
Você possui um histórico organizado de todas as mudanças do projeto.

---

## 🎯 Por que isso é importante no mercado?

Projetos profissionais possuem:

* Centenas ou milhares de arquivos
* Vários programadores trabalhando ao mesmo tempo
* Atualizações constantes
* Correções frequentes de erros

Sem controle de versão, seria praticamente impossível manter organização e segurança.

Por isso, **todo projeto profissional usa Git**.
