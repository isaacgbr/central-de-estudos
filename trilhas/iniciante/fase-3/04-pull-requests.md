# 04 — Pull Requests e Colaboração

Agora que você já entende como projetos são versionados e armazenados na nuvem, vamos aprender como equipes trabalham juntas sem causar bagunça no código.

Em projetos profissionais, várias pessoas modificam o mesmo sistema ao mesmo tempo.

Para organizar isso com segurança, usamos **Pull Requests**.

---

## 🤝 O que é colaboração em projetos?

Colaborar significa que várias pessoas trabalham no mesmo projeto, cada uma desenvolvendo partes diferentes.

Por exemplo:

* Uma pessoa melhora a tela de login
* Outra cria o sistema de pagamentos
* Outra corrige erros no cadastro

Sem organização, alterações poderiam se misturar e causar falhas.

---

## 🔀 O que é um Pull Request?

Pull Request (PR) é um pedido para adicionar suas alterações ao projeto principal.

Funciona assim:

> “Terminei minhas mudanças. Vocês podem revisar antes de juntar ao projeto?”

É um mecanismo de segurança e qualidade.

---

## 🧠 Como funciona um Pull Request (conceito)

1️⃣ Você cria uma branch para sua tarefa
2️⃣ Faz as alterações nela
3️⃣ Envia para o GitHub
4️⃣ Abre um Pull Request
5️⃣ Outras pessoas revisam seu código
6️⃣ Após aprovação, suas alterações entram no projeto principal

Assim, nada é adicionado sem revisão.

---

## 🔍 Por que revisão de código é importante?

A revisão permite:

* Encontrar erros antes que virem problemas
* Melhorar a qualidade do código
* Manter padrão no projeto
* Compartilhar conhecimento entre a equipe

Projetos profissionais valorizam muito esse processo.

---

## 🏢 Como empresas utilizam Pull Requests

Em empresas de tecnologia:

* Nenhuma alteração vai direto para o sistema principal
* Tudo passa por revisão
* Testes são executados antes da aprovação
* Discussões técnicas acontecem nos Pull Requests

Isso evita falhas graves e mantém o sistema estável.

---

## 🎯 Benefícios do Pull Request

✔ Organização das mudanças
✔ Segurança no versionamento
✔ Trabalho em equipe estruturado
