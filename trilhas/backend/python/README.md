## 🌐 O que é Python no backend?

Python é uma linguagem interpretada, de alto nível e com foco em legibilidade. No backend, é usada para construir APIs, automatizar processos e integrar sistemas — especialmente em ambientes onde dados e inteligência artificial também estão presentes.

Com Python no backend você consegue:

- Criar APIs REST rápidas e bem estruturadas
- Automatizar tarefas e processos repetitivos
- Integrar facilmente com bibliotecas de dados e IA
- Construir sistemas de análise e processamento de dados
- Desenvolver microsserviços leves e eficientes

## 📈 Mercado e demanda

- Alta demanda em **startups internacionais e empresas de tecnologia**
- Linguagem dominante em **ciência de dados, machine learning e IA**
- Muito presente em **empresas de produto, fintechs e healthtechs**
- Comunidade enorme com vasto ecossistema de bibliotecas — PyPI com mais de 400 mil pacotes
- Boa porta de entrada para quem vem de outras áreas como matemática, engenharia ou biologia

---

## ✅ Quando escolher Python para o backend?

- O projeto envolve ou vai envolver **dados, machine learning ou IA**
- Você quer uma linguagem com **sintaxe limpa e fácil de aprender**
- O time tem perfis variados — Python é acessível para não-programadores
- Você quer construir **APIs rápidas com FastAPI** ou sistemas simples com Flask
- O projeto é uma **startup, API de médio porte ou sistema de automação**

## ❌ Quando não é a melhor escolha?

- Sistemas que exigem **alta performance e baixa latência** — Python é mais lento que Java e Go
- Projetos com muita **concorrência e paralelismo pesado** — o GIL do Python pode ser um limitante
- Empresas corporativas com cultura forte de Java ou .NET
- Quando a equipe já domina outra linguagem — não vale a troca só pela sintaxe

## 🛠️ Frameworks disponíveis

### Flask
Minimalista e flexível — você decide a estrutura do projeto. Ideal para APIs simples, projetos pequenos e aprender como um servidor web funciona sem muita abstração.

### FastAPI
Moderno, rápido e com suporte nativo a tipagem e documentação automática. Ideal para APIs de médio e grande porte, especialmente quando performance e clareza de contrato são importantes.

> Nesta trilha você vai aprender Python puro primeiro — fundamentos, OOP e estruturas de dados — e só então partir para os frameworks com base sólida.
