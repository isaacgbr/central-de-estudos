## 🌐 O que é JavaScript no backend?

JavaScript nasceu no navegador, mas em 2009 o **Node.js** trouxe o motor V8 do Chrome para fora do browser — permitindo executar JavaScript diretamente no servidor.

Hoje, com Node.js, você consegue:

- Criar APIs REST completas
- Conectar e manipular bancos de dados
- Autenticar usuários com JWT
- Construir sistemas em tempo real com WebSockets
- Automatizar tarefas e criar scripts

## 📈 Mercado e demanda

- Uma das linguagens **mais demandadas no Brasil e no mundo**
- Muito presente em **startups, empresas de produto e freelas**
- Permite ser **fullstack** usando a mesma linguagem no frontend e backend
- Ecossistema gigante — npm com mais de 2 milhões de pacotes disponíveis
- Comunidade enorme — documentação, tutoriais e suporte fácil de encontrar

---

## ✅ Quando escolher JavaScript para o backend?

- Você já conhece JavaScript e quer aproveitar o conhecimento
- O projeto precisa de **respostas rápidas e alta concorrência** (Node.js é excelente nisso)
- Você quer trabalhar como **fullstack** com uma só linguagem
- O time é pequeno ou você está trabalhando sozinho
- O projeto é uma **startup, API de médio porte ou sistema de tempo real**

## ❌ Quando não é a melhor escolha?

- Sistemas com **processamento pesado de CPU** — Node.js é single-thread e não é ideal para isso
- Empresas corporativas que já têm cultura forte de Java ou .NET
- Projetos que exigem **tipagem estática rígida** desde o início — nesse caso considere TypeScript + NestJS

## 🛠️ Frameworks disponíveis

### Express
Minimalista e flexível — você monta a estrutura do jeito que quiser.
Ideal para APIs simples, projetos pequenos e médios, e para aprender como um servidor funciona de verdade.

### NestJS
Estruturado e opinado — segue convenções claras com módulos, controllers e services.
Ideal para sistemas maiores, times com mais de uma pessoa e projetos que precisam escalar.

> Nesta trilha você vai aprender Node.js puro primeiro, entender como tudo funciona por baixo, e depois partir para os frameworks com contexto real de quando e por que usá-los.
